How to Report a Bug

In the course of using technology, bugs happen. Sometimes software does the wrong thing, or doesn't anticipate what you want to do with it.

In this new era of open-source software, it's easy to tell software authors that something's wrong.

Here's what you need to put in your bug report so coders have all of the information they need to help you.

What's Wrong?

It's broken…

If you're reporting a bug, that generally means that something is broken, or wrong about software.

what did you expect to happen? → what happened instead?

So be sure to say not only that something is broken, but how exactly it's broken: does the software give an incorrect result, or no result at all? If the result is incorrect, what did it give, and what did you expect it to give if it worked correctly?

I added 2 + 2 in my calculator.
I expected it to give me a result of 4.
Instead, it gives 5.

System Information

Sometimes software works in one place, for one person, but doesn't somewhere else. Your operating system, browser, and other installed software all has some impact on the bugs that crop up.

Include some basic information about your setup to make issues easier to debug. If it's a web-based application that you use in a browser, include:

  • Browser & Version
  • Browser extensions, if any
  • Operating System

For installed software and libraries, there are some other things to report:

  • Operating System
  • How you installed the software: as a download, from a package manager, or from source
  • Interpreter version, if it's software in a scripting language - are you using Python 2 or 3, or node.js v0.4.0 or v0.6.0?

Be Nice

This is broken - I spent hours debugging it!

Maybe the bug ate your homework, or it's taken you hours to track down the culprit and the bug is something really silly, something that nobody would ever do.

It's okay to be personally annoyed, but be polite on the internet: nobody deserves abuse.
